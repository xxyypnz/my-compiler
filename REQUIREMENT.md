## 一、L26 语言规范

### 1.1 完整文法（EBNF）

```
<program>    ::= <block>
<block>      ::= "{" <decls> <stmts> "}"
<decls>      ::= { <decl> }
<decl>       ::= <type> <ID> ";"
<type>       ::= "int" | "bool" | "set"

<stmts>      ::= { <stmt> }
<stmt>       ::= <assign_stmt>
               | <if_stmt>
               | <while_stmt>
               | <io_stmt>
               | <block>
               | <set_op_stmt>

<assign_stmt> ::= <ID> "=" <expr> ";"
<if_stmt>     ::= "if" "(" <bexpr> ")" <stmt> [ "else" <stmt> ]
<while_stmt>  ::= "while" "(" <bexpr> ")" <stmt>
<io_stmt>     ::= "write" <expr> ";" | "read" <ID> ";"
<set_op_stmt> ::= "add" <ID> <aexpr> ";"
                | "remove" <ID> <aexpr> ";"

<expr>       ::= <bexpr> | <aexpr> | <set_expr>

<aexpr>      ::= <aterm> { ("+" | "-") <aterm> }
<aterm>      ::= <afactor> { ("*" | "/") <afactor> }
<afactor>    ::= <ID> | NUM | "(" <aexpr> ")"

<bexpr>      ::= <bterm> { "||" <bterm> }
<bterm>      ::= <bfactor> { "&&" <bfactor> }
<bfactor>    ::= "true" | "false" | "!" <bfactor>
               | "(" <bexpr> ")"
               | <rel>
               | <set_test>

<rel>        ::= <aexpr> ("<" | "<=" | ">" | ">=" | "==" | "!=") <aexpr>

<set_expr>   ::= "{" [ <aexpr> { "," <aexpr> } ] "}"
               | <ID> "union" <ID>
               | <ID> "inter" <ID>

<set_test>   ::= <aexpr> "in" <ID>
               | "isempty" "(" <ID> ")"
```

### 1.2 词法规范

| 类别 | 说明 |
|------|------|
| 关键字 | `int` `bool` `set` `if` `else` `while` `read` `write` `true` `false` `add` `remove` `union` `inter` `in` `isempty` |
| 标识符 ID | `[a-zA-Z_][a-zA-Z0-9_]*`，不与关键字冲突 |
| 整数 NUM | `[0-9]+` |
| 运算符 | `+` `-` `*` `/` `=` `<` `<=` `>` `>=` `==` `!=` `!` `&&` `\|\|` |
| 分隔符 | `{` `}` `(` `)` `;` `,` |
| 注释 | `//` 行注释，`/* */` 块注释（均忽略） |
| 空白 | 空格、制表符、换行均忽略 |

## 二、语义规范

### 2.1 类型系统

| 类型 | 说明 |
|------|------|
| `int` | 32位整数 |
| `bool` | 布尔值，`true`/`false` |
| `set` | 整数集合，静态实现，最大200个元素，元素互异 |

**类型检查规则：**
- 算术运算（`+` `-` `*` `/`）：操作数必须为 `int`，结果为 `int`
- 关系运算（`<` `<=` `>` `>=` `==` `!=`）：操作数为 `int`，结果为 `bool`
- 逻辑运算（`&&` `||` `!`）：操作数必须为 `bool`，结果为 `bool`
- `add`/`remove`：第一个操作数必须为 `set`，第二个为 `int`
- `union`/`inter`：两个操作数均为 `set`，结果为 `set`
- `in`：左操作数为 `int`，右操作数为 `set`，结果为 `bool`
- `isempty`：操作数为 `set`，结果为 `bool`
- 赋值：左右两侧类型必须一致

### 2.2 作用域规则

- 采用**块作用域**（block scoping），每个 `{ }` 块引入新作用域
- 内层作用域可以**遮蔽**（shadow）外层同名变量
- 查找变量时从当前作用域向外逐层查找
- 块结束时，该块内声明的变量销毁，外层变量恢复可见
- 同一作用域内不允许重复声明同名变量

### 2.3 集合语义

- 集合字面量 `{1, 2, 3}`：元素自动去重，顺序无关
- `add s e`：若 `e` 已在 `s` 中则不操作，否则添加
- `remove s e`：若 `e` 不在 `s` 中则不操作，否则删除
- `union`：返回两集合的并集
- `inter`：返回两集合的交集
- `write` 一个集合时，输出格式为 `{e1, e2, ...}`（元素顺序不作要求）

## 三、类 P-Code 指令集

### 3.1 基础指令（沿用课堂规范）

| 指令 | 操作数 | 说明 |
|------|--------|------|
| `LIT` | `0, n` | 将常数 n 压栈 |
| `LOD` | `l, a` | 将层差 l、偏移 a 处的变量值压栈 |
| `STO` | `l, a` | 将栈顶值存入层差 l、偏移 a 处 |
| `CAL` | `l, a` | 调用过程（本项目暂不需要） |
| `INT` | `0, n` | 栈顶指针增加 n（分配 n 个局部变量空间） |
| `JMP` | `0, a` | 无条件跳转到地址 a |
| `JPC` | `0, a` | 栈顶为 false（0）时跳转到地址 a，弹出栈顶 |
| `OPR` | `0, n` | 执行运算，n 编码见下表 |

### 3.2 OPR 运算编码

| n | 运算 |
|---|------|
| 0 | 过程返回（RET） |
| 1 | 取负（NEG） |
| 2 | 加（ADD） |
| 3 | 减（SUB） |
| 4 | 乘（MUL） |
| 5 | 除（DIV） |
| 6 | 奇偶判断（ODD） |
| 7 | 相等（EQ） |
| 8 | 不等（NEQ） |
| 9 | 小于（LT） |
| 10 | 大于等于（GEQ） |
| 11 | 大于（GT） |
| 12 | 小于等于（LEQ） |
| 13 | 逻辑非（NOT） |
| 14 | 逻辑与（AND） |
| 15 | 逻辑或（OR） |

### 3.3 扩展指令（为支持 set/IO 新增）

| 指令 | 操作数 | 说明 |
|------|--------|------|
| `READ` | `0, 0` | 从标准输入读一个整数，压栈 |
| `WRITE` | `0, 0` | 弹出栈顶整数，输出到标准输出 |
| `WRITES` | `l, a` | 输出层差 l、偏移 a 处的集合 |
| `SET_NEW` | `0, 0` | 在栈上初始化一个空集合（占用固定内存块） |
| `SET_LIT` | `0, n` | 将栈顶 n 个整数弹出，构造集合，结果压栈 |
| `SET_ADD` | `l, a` | 将栈顶整数添加到层差 l、偏移 a 处的集合 |
| `SET_REM` | `l, a` | 将栈顶整数从层差 l、偏移 a 处的集合删除 |
| `SET_IN` | `l, a` | 检查栈顶整数是否在集合中，结果（0/1）压栈 |
| `SET_EMPTY` | `l, a` | 检查集合是否为空，结果（0/1）压栈 |
| `SET_UNION` | `l1,a1,l2,a2` | 计算两集合并集，结果压栈（临时集合） |
| `SET_INTER` | `l1,a1,l2,a2` | 计算两集合交集，结果压栈（临时集合） |
| `SET_COPY` | `l, a` | 将栈顶集合存入层差 l、偏移 a 处 |

> 具体指令编码和操作数格式可在实现时调整，以上为设计参考。

### 3.4 集合的内存表示

集合在运行时栈上占用固定大小的内存块：
- 第0个字：当前元素个数（count）
- 第1~200个字：元素数组（最多200个整数）
- 共 **201个字**（word）的连续空间

`INT 0, n` 分配局部变量时，`set` 类型变量占用 201 个字，`int`/`bool` 占用 1 个字。

## 四、编译器架构

### 4.1 模块划分

```
compile/
├── lexer.l          # Flex 词法规则
├── parser.y         # Bison 语法规则 + 语义动作
├── symtab.c/h       # 符号表（支持嵌套作用域）
├── codegen.c/h      # P-Code 生成
├── pvm.c/h          # 类 P-Code 虚拟机
├── main.c           # 入口：读源文件，调编译器，运行虚拟机
├── Makefile
└── tests/
    ├── test1.l26
    ├── test2.l26
    └── test3.l26
```

### 4.2 编译流程

```
源程序(.l26)
    │
    ▼
[Flex 词法分析] → token 流
    │
    ▼
[Bison 语法分析 + 语义动作]
    ├── 符号表管理（作用域进入/退出、变量查找）
    ├── 类型检查
    └── P-Code 生成
    │
    ▼
P-Code 指令序列
    │
    ▼
[类 P-Code 虚拟机] → 运行输出
```

### 4.3 符号表设计

- 使用**链式作用域栈**：每进入一个 `{ }` 块，压入新的作用域帧
- 每个符号表项记录：变量名、类型、层次（level）、偏移（offset）
- 查找时从栈顶向下遍历，找到第一个匹配项即返回
- 退出块时弹出当前作用域帧

## 十、示例程序参考

### 示例1
```
{
    int val;
    set s;
    s = {1, 2, 3};
    read val;

    if (val in s) {
        add s 10;
        write val;
    }

    while (val > 0) {
        val = val - 1;
        write val;
    }

    write 999;
}
```

### 示例2（嵌套作用域与遮蔽）
```
{
    int x;
    set s;
    x = 10;
    s = {1, 2};

    {
        set x;        // 合法：遮蔽外层 int x
        x = {5, 6};
        add x 7;
        write x;      // 输出 {5, 6, 7}

        write s;      // 合法：向上找到外层 s
    }

    write x;          // 输出 10：内层作用域已销毁
}
```
