## 四、lexer.l 与 parser.y 的连接机制

### 两个文件的分工

```
lexer.l   →  一次吐出一个 token（整数、关键字、标识符…）
parser.y  →  不断问 lexer "下一个 token 是什么" → 拼成语法树 → 执行语义动作
```

接口是 `yylex()`，由 Flex 自动生成，Bison 每需要一个 token 就调用它一次。

### `yylval`：token 附带的数据

光知道 token 类型不够，还需要具体的值（数字是多少、标识符叫什么）。`yylval` 是一个全局 union，lexer 写入，parser 读出。

**`%union` 声明（parser.y:79-94）：**

```c
%union {
    int     ival;     // 存整数值（NUM、TRUE、FALSE）
    char    sval[64]; // 存标识符名字（ID）
    VarType vtype;    // 存类型枚举（int/bool/set）
    struct { VarType type; int level; int offset; } expr;   // 表达式语义值
    struct { int addr; int jpc_idx; PatchNode *list; } ctrl; // 控制流回填信息
    int count;        // expr_list 元素计数
}
```

**lexer.l 对应写法：**

```c
"true"  { yylval.ival = 1; return TRUE_KW; }   // 写 ival，返回 token 类型
"false" { yylval.ival = 0; return FALSE_KW; }

[a-zA-Z_][a-zA-Z0-9_]* {
    strncpy(yylval.sval, yytext, 63);           // 写 sval
    return ID;
}

[0-9]+ {
    yylval.ival = atoi(yytext);                 // 写 ival
    return NUM;
}
```

规律：**lexer 写 `yylval.xxx` 然后 return token类型；parser 用 `$n` 读那个值。**

### token 声明（parser.y:99-106）

```bison
%token <ival> NUM TRUE_KW FALSE_KW   // 携带 ival
%token <sval> ID                      // 携带 sval
%token INT_KW BOOL_KW SET_KW         // 不携带数据，只用于匹配
%token LE GE EQ_OP NE AND_OP OR_OP  // 多字符运算符，不携带数据
```

`<ival>` 告诉 Bison：规则里用 `$n` 引用这个 token 时，自动选 `yylval.ival`。

### 非终结符类型（parser.y:111-114）

```bison
%type <expr>  expr        // expr 的 $$ 是三字段 struct
%type <vtype> type        // type 的 $$ 是 VarType 枚举
%type <count> expr_list   // expr_list 的 $$ 是 int（元素个数）
```

### 优先级（parser.y:119-130，越下越高）

```bison
%nonassoc NO_ELSE    // 解决 dangling-else 歧义的虚拟 token
%nonassoc ELSE
%left OR_OP
%left AND_OP
%right '!'
%nonassoc '<' '>' LE GE EQ_OP NE
%left '+' '-'
%left '*' '/'
%right UMINUS        // 单目负号虚拟 token
%left IN_KW
%left UNION_KW INTER_KW
```

## 五、parser.y 主线文法规则

### `program`（第136-139行）

```bison
program : block { emit(OPR, 0, OPR_RET); }
```

整个程序就是一个 block，结束时 emit RET 指令。

### `block`（第151-167行）

```bison
block
  : '{'
      { scope_enter(); $<ival>$ = emit(INT, 0, 0); }  // 开新作用域，emit占位INT
    decls
      { patch($<ival>2, scope_frame_size()); }         // 知道大小后回填
    stmts
    '}'
      { int sz = scope_frame_size(); scope_exit(); emit(INT, 0, -sz); }
```

三件事：进 `{` 时 emit 占位 `INT`；声明处理完回填真实大小；出 `}` emit `INT 0 -sz` 归还栈空间。

### `decl`（第177-190行）

```bison
decl : type ID ';'
    {
        Symbol *s = sym_declare($2, $1);  // $2=名字, $1=VarType
        if ($1 == T_SET) emit(SET_NEW, ld, s->offset);  // set 变量初始化为空集合
    }
```

### `assign_stmt`（第215-238行）

```bison
assign_stmt : ID '=' expr ';'
    {
        // 类型检查：s->type == $3.type
        int ld = scope_level() - s->level;  // 层差
        if (s->type == T_SET)
            emit(SET_COPY, ld, s->offset);
        else
            emit(STO, ld, s->offset);  // 弹栈顶值存到变量
    }
```

### `if_stmt` / `while_stmt`（第240-287行）：回填模式

```
if:
    [条件表达式]
    JPC 0 ?          ← 条件假跳转，目标未知，先占位
    [then体]
    (patch 回填目标)

while:
loop_top:
    [条件表达式]
    JPC 0 loop_end   ← 条件假则跳出
    [循环体]
    JMP 0 loop_top   ← 无条件跳回
loop_end:
```

### `expr` 中的 `ID`（第364-379行）

```bison
| ID
    {
        Symbol *s = sym_lookup($1);
        $$.type = s->type; $$.level = ld; $$.offset = s->offset;
        if (s->type != T_SET) emit(LOD, ld, s->offset);
        // T_SET 不压栈：集合占201字，地址存在语义值里，供后续指令直接寻址
    }
```

### `expr` 中的算术/逻辑运算（第380-471行）

统一模式：

```bison
| expr '+' expr
    {
        check_int2($1.type, $3.type, "+");  // 类型检查
        $$.type = T_INT; $$.level = 0; $$.offset = 0;
        emit(OPR, 0, OPR_ADD);              // 弹两个数，相加，结果压栈
    }
```

`$1.type` / `$3.type` 是左右子表达式的类型，`$$` 是当前表达式的语义值。

## 七、数据流总结

```
lexer.l                           parser.y
─────────────────────────────────────────────────────
yytext = "42"
yylval.ival = 42      →  token NUM   →  规则里 $n = 42（因为%token<ival> NUM）
return NUM

yytext = "foo"
yylval.sval = "foo"   →  token ID    →  规则里 $n = "foo"（因为%token<sval> ID）
return ID

yytext = "int"
(无 yylval 赋值)      →  token INT_KW →  规则里只用来匹配，不取值
return INT_KW
```
