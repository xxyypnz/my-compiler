# parser.y 深度阅读笔记 06-14-2137

## 一、全局调用链视图

```
yyparse()  ← Bison 生成，main.c 调用的入口
    │
    ├─► program → block
    │               │
    │               ├─► decls → decl → type         （声明链）
    │               │
    │               └─► stmts → stmt                （语句链）
    │                               │
    │                               ├─► assign_stmt
    │                               ├─► if_stmt
    │                               ├─► while_stmt
    │                               ├─► io_stmt
    │                               ├─► set_op_stmt
    │                               └─► block        （递归嵌套）
    │
    └─► expr  ← 被所有 stmt 调用，自递归
                    │
                    ├─► expr_list    （集合字面量内部）
                    ├─► comp_head    （推导式）
                    └─► comp_filter  （推导式）
```

**功能函数（不是文法规则，是序言区定义的C函数）：**
- `make_patch` / `do_patch` / `free_patch`：回填链表管理
- `type_error` / `check_int2` / `check_bool2`：类型检查报错
- `yyerror`：Bison 语法错误回调

## 二、序言区（第1-72行）

`%{ ... %}` 里的内容原样嵌入生成的 `parser.tab.c`，在 `yyparse()` 定义之前。

### 2.1 回填链表（第16-39行）

控制流（if/while）在 emit 跳转指令时，**目标地址还不知道**（还没生成到那里）。回填机制解决这个问题：

```c
typedef struct PatchNode { int addr; struct PatchNode *next; } PatchNode;
```

- `make_patch(addr)`：记录一个"待填坑"的指令地址，返回链表节点
- `do_patch(list, target)`：遍历链表，把每个坑填上真正的目标地址
- `free_patch(list)`：释放链表内存

实际使用时，一个 `PatchNode*` 就代表一条"将来要回填"的跳转指令地址。

### 2.2 类型检查辅助（第44-63行）

```c
type_error(msg)           // 打印行号+报错信息，直接 exit(1)
check_int2(t1, t2, op)   // 要求两个操作数都是 T_INT，否则 type_error
check_bool2(t1, t2, op)  // 要求两个操作数都是 T_BOOL，否则 type_error
```

这三个函数在 `expr` 规则里被大量调用，是语义分析的核心检查点。

### 2.3 全局变量（第70行）

```c
static int g_comp_jpc_end = 0;
```

这是推导式专用的"穿线变量"。`comp_head` 规则生成之后，需要把一个跳转地址传给后续的 `comp_filter` 规则，但 Bison 的语义值传递在这里不方便（两个非终结符之间），所以用全局变量绕过去。

## 三、Bison 声明区（第74-130行）

### 3.1 `%glr-parser`（第74行）

启用 GLR 解析器，允许处理少量局部歧义（例如 `{ expr | ... }` 的 `{` 是集合字面量还是推导式，等读到 `|` 才能确定）。

### 3.2 `%union`（第79-94行）

定义 `yylval` 这个全局 union 能装哪些类型的数据：

| 字段 | 类型 | 用于 |
|------|------|------|
| `ival` | `int` | NUM值、bool字面量值、block中INT指令索引 |
| `sval` | `char[64]` | 标识符名字 |
| `vtype` | `VarType` | type规则的结果（T_INT/T_BOOL/T_SET）|
| `expr` | struct{type,level,offset} | 所有表达式的语义值 |
| `ctrl` | struct{addr,jpc_idx,list} | 控制流（if/while/推导式）的回填信息 |
| `count` | `int` | expr_list 里的元素计数 |

### 3.3 token 和 type 声明（第99-114行）

```bison
%token <ival> NUM TRUE_KW FALSE_KW   // 携带整数值
%token <sval> ID                      // 携带名字字符串
%token INT_KW BOOL_KW ...            // 纯匹配，不携带数据

%type <expr>  expr        // expr 的 $$ 是 union.expr
%type <vtype> type        // type 的 $$ 是 union.vtype
%type <count> expr_list   // expr_list 的 $$ 是 union.count（=ival）
%type <ctrl>  comp_head comp_filter
```

`%token <xxx>` 声明 token 携带哪个 union 字段；`%type <xxx>` 声明非终结符的 `$$` 用哪个字段。没有 `<xxx>` 的 token 只做语法匹配，不取值。

### 3.4 优先级（第119-130行，从低到高）

```bison
%nonassoc NO_ELSE   // 虚拟 token，解决 dangling-else
%nonassoc ELSE
%left OR_OP         // ||
%left AND_OP        // &&
%right '!'
%nonassoc '<' '>' LE GE EQ_OP NE
%left '+' '-'
%left '*' '/'
%right UMINUS       // 虚拟 token，单目负号用 %prec UMINUS 标注
%left IN_KW
%left UNION_KW INTER_KW
```

`NO_ELSE` 和 `UMINUS` 是虚拟 token，从不出现在实际 token 流里，只是优先级锚点：
- `stmt %prec NO_ELSE`：让无 else 的 if 规则优先级低于 ELSE，使 else 优先向内层 if 结合
- `'-' expr %prec UMINUS`：让单目负号比双目 `-` 优先级更高

## 四、文法规则：入口链

### 4.1 `program`（第136-139行）

```bison
program : block { emit(OPR, 0, OPR_RET); }
```

整个源文件就是一个 block。block 结束后 emit 一条 RET 指令，虚拟机看到 RET 就返回，程序结束。这是 P-Code 序列的最后一条指令。

### 4.2 `block`（第150-167行）

```bison
block
  : '{'
      {
          scope_enter();               // 符号表新建作用域帧
          $<ival>$ = emit(INT, 0, 0); // emit 占位 INT，返回该指令的下标
      }
    decls
      {
          patch($<ival>2, scope_frame_size()); // 现在知道了总大小，回填
      }
    stmts
    '}'
      {
          int sz = scope_frame_size();
          scope_exit();                // 符号表弹出作用域帧
          emit(INT, 0, -sz);          // 归还栈空间
      }
```

**关键点：中间动作（mid-rule action）**

Bison 规则里 `{ }` 可以出现在任意位置，不只在末尾。`$<ival>$` 是在 `{` 之后、`decls` 之前插入的动作，它的 `$$` 用 `$<ival>$` 强制指定为 union.ival，存的是 `emit()` 返回的指令下标。

等 `decls` 处理完，`scope_frame_size()` 知道了这个 block 需要多少字，才回填 `patch($<ival>2, ...)` 里的 `2` 是指第2个位置（即那个 mid-rule action 的 `$$`）。

**生成的 P-Code 结构：**
```
INT 0 n        ← 分配 n 个字（n 在 decls 后回填）
SET_NEW ...    ← 每个 set 变量的初始化
[stmts 的代码]
INT 0 -n       ← 归还 n 个字
```

## 五、声明链

### 5.1 `decls`（第172-175行）

```bison
decls : /* empty */ | decls decl
```

左递归，允许零个或多个声明。每个 decl 逐条处理，无需传递值。

### 5.2 `decl`（第177-190行）

```bison
decl : type ID ';'
    {
        Symbol *s = sym_declare($2, $1);  // $2=名字, $1=VarType
        // 重复声明：同一作用域内 sym_declare 返回 NULL
        if ($1 == T_SET) emit(SET_NEW, scope_level() - s->level, s->offset);
    }
```

`sym_declare` 在当前作用域注册变量，同时分配偏移（int/bool 占1字，set 占201字）。

set 变量声明时**必须 emit SET_NEW**：这条指令在运行时把那 201 个字全部清零，否则是垃圾内存。int/bool 不需要，直接 STO 就覆盖了。

### 5.3 `type`（第192-196行）

```bison
type : INT_KW  { $$ = T_INT; }
     | BOOL_KW { $$ = T_BOOL; }
     | SET_KW  { $$ = T_SET; }
```

纯粹把关键字 token 转换为枚举值，作为 `$$` 向上传递给 `decl` 的 `$1`。

## 六、语句链

### 6.1 `stmts` / `stmt`（第201-213行）

```bison
stmts : /* empty */ | stmts stmt   // 零个或多个语句，左递归
stmt  : assign_stmt | if_stmt | while_stmt | io_stmt | block | set_op_stmt
```

`stmt` 是分发节点，本身不做任何事，只选择走哪条分支。

### 6.2 `assign_stmt`（第215-238行）

```bison
assign_stmt : ID '=' expr ';'
    {
        Symbol *s = sym_lookup($1);         // 查符号表，必须已声明
        // 类型检查：s->type == $3.type
        int ld = scope_level() - s->level;  // 层差：当前层 - 声明所在层
        if (s->type == T_SET)
            emit(SET_COPY, ld, s->offset);  // set：从 temp_set 拷贝过去
        else
            emit(STO, ld, s->offset);       // int/bool：弹栈顶存入变量
    }
```

**层差（ld）的含义**：每个 block 是一个"层"。变量在第2层声明，现在在第4层使用，ld=2，虚拟机用它找到正确的帧基址。

**set 赋值为何用 SET_COPY 而不是 STO**：set 在 expr 里的结果不在栈上，而是在 `temp_set`（全局临时缓冲区）里。SET_COPY 把 temp_set 的内容复制到目标变量地址。

### 6.3 `if_stmt`（第240-270行）

两个规则分支：无 else 和有 else，都用回填实现。

**无 else 分支：**
```
[条件 expr]       ← 执行后栈顶是 0 或 1
JPC 0 ?          ← 条件为0跳转，目标未知，emit返回下标记录在 jpc_idx
[then 体]
(patch jpc_idx → current_addr)   ← then 结束后回填，指向 then 之后
```

**有 else 分支：**
```
[条件 expr]
JPC 0 ?          ← jpc_idx，then 假时跳走
[then 体]
JMP 0 ?          ← jmp_idx，then 执行完跳过 else
(patch jpc_idx → current_addr)   ← 指向 else 开始
[else 体]
(patch jmp_idx → current_addr)   ← 指向 else 之后
```

`$<ctrl>5` 和 `$<ctrl>7` 的数字是 Bison 规则里的位置编号（第5个符号/第7个符号）。

### 6.4 `while_stmt`（第272-287行）

```bison
while_stmt
  : WHILE
      { $<ctrl>$.addr = current_addr(); }  // 记住循环顶部地址
    '(' expr ')'
      {
          // 类型检查：必须是 bool
          $<ctrl>$.jpc_idx = emit(JPC, 0, 0);  // 条件假时跳出
          $<ctrl>$.list    = make_patch($<ctrl>$.jpc_idx);
      }
    stmt
      {
          emit(JMP, 0, $<ctrl>2.addr);          // 跳回循环顶
          do_patch($<ctrl>6.list, current_addr()); // 回填 JPC 目标
      }
```

**生成的 P-Code 结构：**
```
loop_top:
    [条件 expr]
    JPC 0 loop_end    ← 条件为0跳出
    [循环体]
    JMP 0 loop_top    ← 无条件跳回
loop_end:
```

`$<ctrl>2.addr` 是 WHILE 之后的 mid-rule action 记录的 `loop_top`（用 `$<ctrl>2` 因为那是第2个位置的 mid-rule action 的 `$$`）。

### 6.5 `io_stmt`（第289-317行）

**write：**
```bison
WRITE_KW expr ';'
    {
        if ($2.type == T_SET) {
            if ($2.level >= 0)   emit(WRITES, $2.level, $2.offset); // 命名 set 变量
            else                 emit(WRITET, 0, 0);                  // 临时结果（字面量/union/inter）
        } else {
            emit(WRITE, 0, 0);  // 弹栈顶整数/布尔，输出
        }
    }
```

`$2.level` 的约定：
- `>= 0`：expr 是一个具名 set 变量，level 是层差，offset 是偏移，可以直接寻址
- `== -1`：set 字面量 `{1,2,3}`，结果在 temp_set
- `== -2`：union/inter/推导式，结果在 temp_set

**read：**
```bison
READ_KW ID ';'
    {
        // 只允许读 int 变量
        emit(READ, 0, 0);                          // 从 stdin 读一个整数压栈
        emit(STO, scope_level() - s->level, s->offset); // 存入变量
    }
```

### 6.6 `set_op_stmt`（第319-342行）

```bison
ADD_KW ID expr ';'
    {
        // ID 必须是 set，expr 必须是 int
        emit(SET_ADD, scope_level() - s->level, s->offset);  // 弹栈顶 int，加入集合
    }

REMOVE_KW ID expr ';'
    {
        emit(SET_REM, scope_level() - s->level, s->offset);  // 弹栈顶 int，从集合删除
    }
```

两条语句都是先由 `expr` 把元素值压栈，再由 `SET_ADD`/`SET_REM` 弹出并操作集合。

## 七、表达式系统（expr，第347-617行）

`expr` 是整个 parser.y 里最长的规则，有约20个分支。所有分支共同约定：**执行后，`$$`（union.expr）记录 {type, level, offset}**，同时按照类型决定是否在栈上留下值。

### 7.1 `$$` 三字段的含义

| 字段 | 含义 |
|------|------|
| `type` | 表达式结果类型（T_INT / T_BOOL / T_SET） |
| `level` | 仅对 set 有意义：`>=0` 表示命名变量的层差，`-1` 表示字面量结果在 temp_set，`-2` 表示运算结果在 temp_set |
| `offset` | 仅对命名 set 变量有意义：变量在帧中的偏移 |

int/bool 的 level 和 offset 填 0，因为它们的值已经在栈顶，不需要再寻址。

### 7.2 字面量分支（第348-362行）

```bison
NUM       { $$.type = T_INT;  ...; emit(LIT, 0, $1); }    // $1 = 数字值
TRUE_KW   { $$.type = T_BOOL; ...; emit(LIT, 0, 1); }
FALSE_KW  { $$.type = T_BOOL; ...; emit(LIT, 0, 0); }
```

直接把值压栈。

### 7.3 ID 分支（第363-379行）

```bison
| ID
    {
        Symbol *s = sym_lookup($1);
        int ld = scope_level() - s->level;
        $$.type = s->type; $$.level = ld; $$.offset = s->offset;
        if (s->type != T_SET) emit(LOD, ld, s->offset);
        // T_SET 不 emit：集合太大（201字），不放栈上，地址存在 $$ 里
    }
```

**set 变量为何不压栈**：LOD 只压一个 word，但 set 需要 201 个 word。运行时的集合操作指令（SET_ADD、SET_IN 等）接收层差+偏移直接寻址，不需要把 set 搬到栈上。

### 7.4 算术运算分支（第380-409行）

统一模式：
```bison
| expr '+' expr
    {
        check_int2($1.type, $3.type, "+");  // 两侧必须是 int
        $$.type = T_INT; $$.level = 0; $$.offset = 0;
        emit(OPR, 0, OPR_ADD);             // 弹两个值相加，结果压栈
    }
```

`$1` 和 `$3` 分别是左右子 expr 的 `$$`（`union.expr`），`.type` 就是子表达式的类型。减/乘/除同理。

单目负号（第404-409行）：
```bison
| '-' expr %prec UMINUS
    {
        if ($2.type != T_INT) type_error("unary minus requires int");
        emit(OPR, 0, OPR_NEG);
    }
```

`%prec UMINUS` 告诉 Bison 这条规则的优先级按 UMINUS 算（比双目 `-` 高），否则 `-3` 会被误解析为 `(0) - 3`。

### 7.5 关系运算分支（第410-447行）

`<` `>` `<=` `>=` `!=` 模式相同：两侧要求 int，结果是 bool，emit 对应 OPR。

`==` 特殊（第434-447行）：
```bison
| expr EQ_OP expr
    {
        if ($1.type == T_SET && $3.type == T_SET)
            emit(SET_EQL, ENCODE2($1.level, $1.offset), ENCODE2($3.level, $3.offset));
            // 加分项：set 相等判断，直接寻址两个 set，不经过栈
        else if ($1.type == $3.type)
            emit(OPR, 0, OPR_EQ);   // int/bool 常规相等
        else
            type_error("'==' operands must have the same type");
    }
```

### 7.6 逻辑运算分支（第454-471行）

`&&` `||` 要求两侧 bool，`!` 要求单个 bool，全部通过 OPR 指令实现。

### 7.7 括号（第472-473行）

```bison
| '(' expr ')' { $$ = $2; }
```

直接透传子表达式的语义值，不 emit 任何指令。

### 7.8 集合字面量（第475-485行）

```bison
| '{' expr_list '}'
    {
        $$.type = T_SET; $$.level = -1; $$.offset = $2; // $2 是元素个数
        emit(SET_LIT, 0, $2);  // 弹出 $2 个栈上的 int，构造集合放入 temp_set
    }
| '{' '}'
    {
        $$.type = T_SET; $$.level = -1; $$.offset = 0;
        emit(SET_LIT, 0, 0);   // 空集合
    }
```

`expr_list` 中每个元素表达式都已经把值压栈，`SET_LIT` 一次性弹出 n 个，去重后存入 `temp_set`。level=-1 是标记"结果在 temp_set，且是字面量来源"。

### 7.9 set union / inter（第487-523行）

```bison
| ID UNION_KW ID
    {
        // 两侧均必须是 set 变量
        emit(SET_UNION, ENCODE2(ld1, s1->offset), ENCODE2(ld3, s3->offset));
        $$.type = T_SET; $$.level = -2; // 结果在 temp_set
    }
```

`ENCODE2(ld, off)` 把层差和偏移编码进一个整数（`ld * 10000 + off`），因为 SET_UNION 指令需要同时传两个变量的地址，而指令结构只有 `l` 和 `a` 两个字段。

level=-2 和 level=-1 都表示结果在 temp_set，用不同值只是为了来源可追溯（对运行时无影响，两者在 `io_stmt` 里都走 `emit(WRITET)` 分支）。

### 7.10 set 成员检查（第525-548行）

```bison
| expr IN_KW ID
    {
        // expr 类型必须是 int，ID 必须是 set
        emit(SET_IN, scope_level() - s->level, s->offset); // 弹栈顶 int，检查是否在集合里
        $$.type = T_BOOL;
    }

| ISEMPTY_KW '(' ID ')'
    {
        emit(SET_EMPTY, scope_level() - s->level, s->offset); // 检查 count==0
        $$.type = T_BOOL;
    }
```

## 八、推导式（扩展功能，第577-696行）

语法：`{ body_expr | ID in SRC_ID if filter_expr }`

例：`{ x + 1 | x in s if x > 2 }` —— 取 s 中大于2的元素，每个加1后组成新集合

规则分成三段，按解析顺序依次执行：

### 8.1 `comp_head`（第624-681行）：`ID in SRC_ID`

这是推导式语法里最先被解析的部分（在 `|` 之前），它做了大量初始化工作：

```
1. 查找源集合 SRC_ID（必须已声明为 set）
2. scope_enter()：推导式有独立作用域
3. emit INT 0 0（占位）
4. sym_declare("__comp_tmp", T_SET)  ← 隐藏变量：收集结果的临时集合
5. sym_declare("__comp_idx", T_INT)  ← 隐藏变量：循环下标（0, 1, 2, ...）
6. sym_declare(ID, T_INT)            ← 迭代变量（用户写的那个 x）
7. patch INT 占位 → 实际帧大小
8. emit SET_NEW（初始化 __comp_tmp 为空集合）
9. emit LIT 0 + STO（__comp_idx = 0）
10. 记录 loop_top = current_addr()
11. emit LOD __comp_idx + LOD src.count + OPR LT → 循环条件
12. emit JPC 0 ?（jpc_end，循环结束跳转，目标未知）← 存入 g_comp_jpc_end
13. emit LOD __comp_idx + SET_ELEM + STO → 取第 idx 个元素赋给迭代变量
```

`$$` 打包返回：`addr = tmp_off`，`jpc_idx = idx_off`，`list = loop_top`（用 intptr_t 转指针存）。

**SET_ELEM 指令**：弹出栈顶的下标 n，返回集合第 n 个元素（从0计数）。这是这里专门为推导式新增的指令。

### 8.2 中间的 `expr`（第577行 `'{' comp_head expr comp_filter '}'` 中的第三个符号）

这就是普通的 expr 规则，但它在 `comp_head` 之后才被解析，此时推导式的内部作用域已经建好，迭代变量已声明，所以 expr 里可以用 `x`（迭代变量）。

### 8.3 `comp_filter`（第688-696行）：`if filter_expr`

```bison
comp_filter : IF expr
    {
        // expr 必须是 bool（过滤条件）
        $$.addr    = g_comp_jpc_end;          // 从全局变量拿回 jpc_end
        $$.jpc_idx = emit(JPC, 0, 0);         // jpc_skip：过滤条件假时跳过 body
    }
```

### 8.4 推导式收尾动作（第578-616行）

`'{' comp_head expr comp_filter '}'` 的最终 action，按顺序 emit：

```
emit SET_ADD 0 tmp_off          ← 把 body_expr 的结果（栈顶）加入 __comp_tmp
patch(jpc_skip → current_addr)  ← 过滤条件假时跳到这里（跳过 SET_ADD）
emit LOD idx + LIT 1 + OPR ADD + STO idx  ← __comp_idx++
emit JMP loop_top               ← 跳回循环顶
patch(jpc_end → current_addr)  ← 循环结束后到这里
emit SET_UNION(__comp_tmp, __comp_tmp)  ← 把 __comp_tmp 复制到 temp_set（作用域销毁前）
int sz = scope_frame_size()
scope_exit()
emit INT 0 -sz                  ← 销毁推导式内部作用域
```

最后 `$$.level = -2`，表示结果在 temp_set。

**为什么用 SET_UNION(tmp, tmp) 来复制**：直接调用 set_union(temp, a, a) 效果等同于 set_copy，但复用了已有指令，避免新增 SET_COPY_TO_TEMP 指令。

**生成的完整 P-Code 结构：**
```
INT 0 n             ← 推导式作用域帧
SET_NEW tmp         ← 初始化结果集合
LIT 0; STO idx      ← idx = 0
loop_top:
  LOD idx
  LOD src.count
  OPR LT
  JPC 0 loop_end    ← idx >= count 跳出
  LOD idx
  SET_ELEM src      ← x = src[idx]
  STO iter
  [filter expr]
  JPC 0 skip        ← 过滤条件假时跳过 body
  [body expr]
  SET_ADD tmp       ← 加入结果集合
skip:
  LOD idx; LIT 1; OPR ADD; STO idx  ← idx++
  JMP loop_top
loop_end:
  SET_UNION tmp tmp ← 结果拷贝到 temp_set
  INT 0 -n          ← 销毁帧
```

## 九、expr_list（第701-706行）

```bison
expr_list : expr              { $$ = 1; }
          | expr_list ',' expr { $$ = $1 + 1; }
```

左递归，每个 `expr` 已经把值压栈了。`$$` 只计数，最后把计数传给 `SET_LIT 0 n` 的 `n`，告诉虚拟机要弹几个值。

## 十、错误处理（第710-713行）

```c
void yyerror(const char *s) {
    fprintf(stderr, "parse error at line %d: %s\n", yylineno, s);
    exit(1);
}
```

Bison 自动调用这个函数报告语法错误（token 不匹配时）。`yylineno` 由 Flex 自动维护（因为 `lexer.l` 里有 `%option yylineno`）。语义错误则在各规则的 action 里手动 `fprintf + exit(1)`。

## 十一、关键设计决策汇总

| 问题 | 解决方案 |
|------|---------|
| 跳转目标未知 | 回填：先 emit 占位，事后 patch |
| block 分配大小未知 | mid-rule action emit 占位 INT，decls 后 patch |
| set 不适合放栈上 | ID规则不 emit LOD，地址存在 `$$` 里，指令直接寻址 |
| temp_set 的 level 区分 | -1=字面量，-2=运算结果，≥0=命名变量（`write`靠此决定指令） |
| 推导式跨规则传 jpc_end | 全局变量 `g_comp_jpc_end`（Bison 语义值传递此处受限） |
| SET_UNION 两个四元参数 | `ENCODE2(ld, off)` = ld×10000+off 压缩进 l/a 两个字段 |
| 推导式结果在作用域销毁前逃逸 | 销毁前 emit SET_UNION(tmp,tmp) 把结果搬到 temp_set |
