# L26 编译器设计说明

## 1. 项目目标

本项目实现 L26 语言编译器。编译器使用 Flex 完成词法分析，使用 Bison 完成语法分析、语义检查和 P-Code 生成，最后由内置类 P-Code 虚拟机执行生成的指令序列。

实现目标包括：

- 支持 `int`、`bool`、`set` 三种类型。
- 支持块作用域、变量声明、变量遮蔽和同层重复声明检查。
- 支持赋值、算术表达式、关系表达式、逻辑表达式。
- 支持 `if/else`、`while`、`read`、`write`。
- 支持集合字面量、`add`、`remove`、`in`、`isempty`、`union`、`inter`。
- 生成类 P-Code 指令，并由虚拟机执行。
- 实现加分项：P-Code 可视化和单步运行、集合相等、集合推导式。

## 2. 扩展后文法

基础文法：

```ebnf
<program>    ::= <block>
<block>      ::= "{" <decls> <stmts> "}"
<decls>      ::= { <decl> }
<decl>       ::= <type> <ID> ";"
<type>       ::= "int" | "bool" | "set"

<stmts>      ::= { <stmt> }
<stmt>       ::= <assign_stmt>
               | <if_stmt>
               | <while_stmt>
               | <io_stmt>
               | <block>
               | <set_op_stmt>

<assign_stmt> ::= <ID> "=" <expr> ";"
<if_stmt>     ::= "if" "(" <expr> ")" <stmt> [ "else" <stmt> ]
<while_stmt>  ::= "while" "(" <expr> ")" <stmt>
<io_stmt>     ::= "write" <expr> ";" | "read" <ID> ";"
<set_op_stmt> ::= "add" <ID> <expr> ";"
                | "remove" <ID> <expr> ";"

<expr>       ::= NUM | TRUE | FALSE | ID
               | <expr> "+" <expr>
               | <expr> "-" <expr>
               | <expr> "*" <expr>
               | <expr> "/" <expr>
               | "-" <expr>
               | <expr> "<"  <expr>
               | <expr> "<=" <expr>
               | <expr> ">"  <expr>
               | <expr> ">=" <expr>
               | <expr> "==" <expr>
               | <expr> "!=" <expr>
               | <expr> "&&" <expr>
               | <expr> "||" <expr>
               | "!" <expr>
               | "(" <expr> ")"
               | "{" [ <aexpr> { "," <aexpr> } ] "}"
               | <ID> "union" <ID>
               | <ID> "inter" <ID>
               | <expr> "in" <ID>
               | "isempty" "(" <ID> ")"
```

加分项扩展：

```ebnf
<set_equality>      ::= <ID> "==" <ID>       (* 两侧均为 set 类型变量 *)
<set_comprehension> ::= "{" <aexpr> "|" <ID> "in" <ID> "if" <expr> "}"
```

说明：

- `set == set` 只支持集合变量之间比较，例如 `s1 == s2`。
- `set != set` 不支持，编译期报错。
- 非变量集合相等，例如 `{1} == {1}`，编译期报错。
- 集合推导式中的迭代变量只在推导式内部作用域有效。

## 3. 代码结构

| 文件 | 作用 |
|------|------|
| `lexer.l` | Flex 词法规则，识别关键字、标识符、数字、运算符、分隔符和注释。 |
| `parser.y` | Bison 语法规则、类型检查、作用域管理和 P-Code 生成。 |
| `symtab.c/h` | 链式作用域符号表，支持嵌套块和变量遮蔽。 |
| `codegen.c/h` | P-Code 指令缓冲、发射、回填和可视化打印。 |
| `pcode.h` | P-Code 指令枚举、OPR 子操作、集合大小和编码宏。 |
| `pvm.c/h` | 类 P-Code 虚拟机，执行算术、控制流、IO 和集合指令。 |
| `main.c` | 命令行入口，支持普通运行、单步模式和只打印 P-Code。 |
| `Makefile` | 构建、正向测试、负向测试和完整测试入口。 |
| `tests/` | 正向测试程序。 |
| `tests/errors/` | 负向测试程序。 |
| `scripts/run_error_tests.sh` | 负向测试批处理脚本。 |

## 4. 编译流程

```text
L26 源程序
  -> Flex 词法分析
  -> Bison 语法分析
  -> 语义检查
  -> P-Code 生成
  -> P-Code 可视化打印
  -> 虚拟机执行
  -> 输出结果
```

## 5. 符号表与作用域

符号表采用作用域链结构。每进入一个 `{}` 块，编译器调用 `scope_enter()` 新建作用域；每离开一个块，调用 `scope_exit()` 删除当前作用域。

每个符号记录：

- 名称
- 类型
- 所在嵌套层次
- 在当前运行时帧中的偏移

同一作用域内重复声明会报错。内层作用域可以声明同名变量遮蔽外层变量。变量查找从当前作用域向外层逐级查找。

## 6. 类型检查

主要类型规则：

- `+ - * /` 需要两个 `int` 操作数，结果为 `int`。
- `< <= > >=` 需要两个 `int` 操作数，结果为 `bool`。
- `== !=` 需要两侧类型一致；其中 `set == set` 只允许集合变量，`set != set` 不支持。
- `&& || !` 需要 `bool` 操作数。
- `add/remove` 第一个操作数必须是 `set` 变量，第二个操作数必须是 `int`。
- `union/inter` 两侧必须是 `set` 变量。
- `in` 左侧必须是 `int`，右侧必须是 `set` 变量。
- `isempty` 参数必须是 `set` 变量。
- 赋值左右类型必须一致。

## 7. P-Code 与虚拟机设计

基础指令包括：

- `LIT`：压入常量。
- `LOD`：读取变量。
- `STO`：保存变量。
- `INT`：进入或退出块帧。
- `JMP`：无条件跳转。
- `JPC`：条件跳转。
- `OPR`：执行算术、关系和逻辑运算。

集合扩展指令包括：

- `SET_NEW`：初始化空集合。
- `SET_LIT`：构造集合字面量。
- `SET_ADD`、`SET_REM`：添加或删除集合元素。
- `SET_IN`、`SET_EMPTY`：集合成员和空集判断。
- `SET_UNION`、`SET_INTER`：集合并集和交集。
- `SET_COPY`：复制临时集合到集合变量。
- `SET_EQL`：集合相等判断。
- `SET_ELEM`：按索引读取集合元素，用于集合推导式遍历。
- `WRITES`、`WRITET`：输出集合变量或临时集合。

集合运行时表示为固定 201 个整数：

- 第 0 个整数为元素个数。
- 第 1 到第 200 个整数保存元素。
- 插入时自动去重。
- 输出时排序，保证显示稳定。

虚拟机使用整数数组作为运行栈，并用 `frame_base[]` 维护嵌套块帧基址。每个块进入时执行 `INT 0 n`，退出时执行 `INT 0 -n`。

## 8. 加分项实现

### 8.1 P-Code 可视化和单步运行

每次编译后都会打印完整 P-Code 列表，显示地址、指令名和操作数。`OPR` 指令会附加子操作说明。

运行：

```bash
./l26c -s tests/test1.l26
```

单步模式支持：

- Enter：执行一条指令。
- `r`：切换到连续运行。
- `q`：退出。

### 8.2 集合相等

语法：

```l26
if (s1 == s2) {
    write 1;
}
```

编译器生成 `SET_EQL` 指令。虚拟机比较元素个数和元素集合，顺序无关。

### 8.3 集合推导式

语法：

```l26
result = { x * 2 | x in s1 if x > 2 };
```

实现方式：

1. 为推导式建立独立作用域。
2. 创建隐藏临时集合、索引变量和迭代变量。
3. 遍历源集合中的每个元素。
4. 将当前元素写入迭代变量。
5. 判断过滤条件。
6. 条件为真时计算结果表达式并加入临时集合。
7. 循环结束后将临时集合复制到目标集合变量。

```bash
./l26c -s tests/test1.l26

```

```bash
make test-errors
```

```bash
make test-all
```
